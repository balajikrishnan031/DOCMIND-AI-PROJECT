2. Introduction to Python, AIML & RAG

A. Python Ecosystem in AIML

Python has emerged as the industry standard language for Artificial Intelligence and Machine Learning due to its clear syntax, readability, and extensive libraries. Key modules implemented in the DocMind AI platform include:

  • Numpy: Used for fast vector mathematical computations, matrix norms, and dot products for cosine similarity calculation.
  • Sentence-Transformers: Utilizes pre-trained deep learning encoder transformers (specifically `all-MiniLM-L6-v2`) to turn raw text strings into dense 384-dimensional arrays.
  • Flask: Acts as the primary lightweight WSGI web framework to route server events and service Jinja HTML templates.
  • PDFPlumber / Extractor: Used for structural text extraction and locating precise page positions inside dense textbook layout streams.

B. Understanding Retrieval-Augmented Generation (RAG)

Large Language Models (LLMs) suffer from hallucinations and lack domain context. A RAG pipeline addresses this by splitting documents into overlapping paragraphs, embedding them, indexing them in a vector database, and matching the user query with relevant context blocks before sending the prompt to the inference model.

3. Problem Statement & Objectives

A. Problem Statement

Standard keyword-based study searches (e.g., Ctrl+F) fail when querying conceptual definitions (e.g., searching for "CPU allocation latency" does not match pages discussing "Round Robin wait times" unless the exact keywords match). Furthermore, students struggle to align massive syllabus structures (with complex units) to exact textbook pages. Active recall methods like flashcard creation are time-consuming to execute manually.

B. Project Objectives

DocMind AI was developed during this internship to solve these limitations through the following engineering objectives:

  1. Semantic Textbook Indexing: Process dense academic documents and build high-precision vector maps.
  2. Unit-Syllabus Mapping: Match unit syllabus descriptions to document content with page-level citations.
  3. Automated Active Recall: Generate 3D study flashcards and practice quizzes mapped to Bloom's taxonomy.
  4. Multi-Level Architecture: Provide customizable portals for Standard 1-12 students, college professors, and academic deans.

5. Implemented Modules

Module 1: Document Extractor & Chunker

`DocumentExtractor` uses PDFPlumber to scan file layouts page-by-page, capturing headers and tables. `TextChunker` groups pages into chunks of 1000 characters with a 150-character overlap to preserve semantic flow.

Module 2: Sentence-Transformers & Vector Storage

`Embedder` encodes text blocks into 384-dimensional dense vectors. `VectorStore` persists these embeddings locally in JSON format, utilizing Numpy for cosine calculations.

Module 3: Syllabus Parser & Matcher

Integrates AI models to parse unstructured unit syllabus entries, mapping individual topics to textbook page ranges dynamically.

Module 4: Bloom's Quiz & Scoreboard

Automatically populates assessment items into cognitive buckets (Remembering, Applying, Analyzing) to test student knowledge, logging scores on a visual dashboard scoreboard.

6. Code Implementation & Mathematics

A. Cosine Similarity Matching

The mathematical formula used to determine how close a document paragraph is to a student's search question is defined by:

$$\text{Cosine Similarity} = \frac{\sum_{i=1}^{N} A_i B_i}{\sqrt{\sum_{i=1}^{N} A_i^2} \sqrt{\sum_{i=1}^{N} B_i^2}}$$

Here is the actual Python logic implemented in `backend/utils/vector_store.py` that utilizes Numpy-optimized dot products for high-speed calculation: 

# Convert vectors to numpy array
vectors = np.array([item["vector"] for item in index_data], dtype=np.float32)
q_vec = np.array(query_vector, dtype=np.float32)

# Calculate dot products and norms
dot_products = np.dot(vectors, q_vec)
vectors_norms = np.linalg.norm(vectors, axis=1)
q_norm = np.linalg.norm(q_vec)

# Cosine Similarity calculation
similarities = dot_products / (vectors_norms * q_norm + 1e-8)
